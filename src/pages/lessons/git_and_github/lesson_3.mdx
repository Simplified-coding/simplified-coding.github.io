<h2 style="text-align: center;"> <span style={{ color: 'purple', fontWeight: 'bold' }}> Δημιουργία Branch και Αλλαγή Branch </span> </h2>

Τα κλαδιά (branches) είναι διαίρεση του κώδικα, όπου μπορούμε να πειραματιστούμε και να κάνουμε αλλαγές χωρίς να επηρεάζουμε τον κώδικα στο κύριο κλαδί (main/master branch).
Τα κλαδιά είναι χρήσιμα όταν θέλουμε να δημιουργήσουμε μια νέα λειτουργία ή να διορθώσουμε ένα σφάλμα χωρίς να επηρεάσουμε τον κώδικα στο κύριο κλαδί.

Για να δημιουργήσετε και να αλλάξετε μεταξύ κλαδιών στο Git, ακολουθήστε τα εξής βήματα:

1. **Δημιουργία Νέου Κλαδιού:**
Για να δημιουργήσετε ένα νέο κλαδί, χρησιμοποιήστε την ακόλουθη εντολή:

```bash
git branch <όνομα_κλαδιού>
   ```

Αυτή η εντολή δημιουργεί ένα νέο κλαδί.

2. **Αλλαγή στο Νέο Κλαδί:**
Για να πάτε στο νέο κλαδί που δημιουργήσατε, χρησιμοποιήστε την ακόλουθη εντολή:

```bash
git checkout <όνομα_κλαδιού>
   ```

Αυτή η εντολή θα σας μεταφέρει στο επιλεγμένο κλαδί.

3. **Εμφάνιση των Κλαδιών:**
Για να δείτε μια λίστα των κλαδιών και να ελέγξετε ποιο κλαδί είναι το τρέχον, χρησιμοποιήστε την ακόλουθη εντολή:

```bash
git branch
   ```

Το κλαδί με το αστεράκι (*) δίπλα από το όνομά του είναι το ενεργό κλαδί.

<h2 style="text-align: center;"> <span style={{ color: 'purple', fontWeight: 'bold' }}> GitHub pull requests </span> </h2>

Τα GitHub pull requests επιτρέπουν τη σύνδεση και συνεργασία μεταξύ διαφορετικών κλαδιών στο repository σας στο GitHub.
Οι προγραμματιστές χρησιμοποιούν τα pull requests για να προτείνουν τις αλλαγές τους στον κώδικα και να συζητήσουν για τις τροποποιήσεις πριν ενσωματωθούν (merge) στο κύριο branch του repository.

Ακολουθήστε αυτά τα βήματα για να δημιουργήσετε ένα GitHub pull request:

1. Μεταβείτε στη σελίδα του repository στο GitHub.
2. Κάντε commit και push τις αλλαγές σας στο GitHub στο νέο branch που δημιουργήσατε.
3. Κάντε κλικ στο κουμπί "New pull request".
4. Επιλέξτε το branch στο οποίο θέλετε να κάνετε merge τις αλλαγές σας (συνήθως το κύριο κλαδί).
5. Δημιουργήστε το pull request.
6. Άλλοι συνεργάτες μπορούν να αναθεωρήσουν τις αλλαγές σας, να σχολιάσουν και να προτείνουν αλλαγές πριν το merge.
7. Μπορείτε να συζητήσετε και να διερευνήσετε οποιεσδήποτε αντιρρήσεις ή βελτιώσεις.
8. Όταν ολοκληρωθεί η αναθεώρηση και οι αλλαγές είναι έτοιμες, μπορείτε να ενσωματώσετε τις αλλαγές σας στο κύριο branch μέσω του pull request.
